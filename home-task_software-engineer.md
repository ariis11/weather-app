# Software Engineer Home Task

## Objective

Build a web application that displays weather information for a selected location.

The goal of this task is to evaluate your ability to design and implement a small but complete product, make pragmatic technical decisions, and document your work clearly.

## Scenario

Create a website where a user can search for a location, select it using autocomplete, and view the current weather information for that location.

The application should also remember previously searched locations using a persistence mechanism of your choice.

## Technical Requirements

- Use `Node.js`.
- Use `TypeScript`.
- Use any weather API of your choice.
- Provide a location input with autocomplete support.
- Persist search history in storage of your choice.
- Include AI prompt history used during development.
- Share your solution in a Git repository with commit history that reflects the work done.

## Weather Information

At minimum, display:

- location name
- current temperature
- weather condition summary

## Bonus Points

The following are not required, but will be considered positively:

- automated tests
- robust error handling
- loading and empty states
- responsive UI
- clean commit history
- thoughtful project structure

## Submission

Please submit:

1. A link to the Git repository.
2. A short note describing how to run the project locally.
3. Any assumptions, tradeoffs, or known limitations.

## Deadline

The deadline is 1 week from receiving this file.
